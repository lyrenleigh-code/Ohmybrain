- **日期**：2026-06-25 ingest
- **类型**：文档方法论模板（reference）
- **适用**：[[uwacomm]] 体系外的 **DocProcess 方案 / 论证 / 报价类项目**（UWAprojDoc / DigitalTwinGuide / Pricing 等）撰写军队 / 科研院所**设施建设项目立项论证报告**时的骨架与口径参照

## 核心观点

### 1. 立项论证报告 18 节骨架（"总—分"逐层论证）

1. 项目基本情况
2. 国内外建设情况（国外 / 国内 / 发展趋势 三段，每段"XX 机构—XX 年—建设内容—典型指标—功能—效益"句式）
3. 建设必要性
4. 建设需求
5. 建设目标
6. **建设方案**（核心，见下 §2）
7. 建设周期及进度安排
8. 经费概算（核心，见下 §3）
9. 项目组成员及分工
10. 可行性分析
11. 运行模式分析
12. 综合性要求
13. 预期效益
14. 验收方案
15. 现有科研资产利旧情况
16. 特殊基建需求
17. 监管措施
18. 需要说明的其它问题

### 2. 建设方案章 = "总方案 + 主要建设内容"

- **总方案**：①总体思路与原则（指导思想 / 建设原则「统筹规划·体系设计·整合资源」/ 总体架构「一主三分·一体两翼」可图表化 / 各部分关系 / 创新点）②总体技术路线（自主可控 + 进口集成融合）③总体技术指标（量化为主、定性为辅，系统性 / 先进性 / 可考核性）④技术先进性（图表对比）。
- **主要建设内容**：按 **"是什么（作用内涵）— 有什么（建设内容组成）— 怎么建（标准规范 / 技术手段 / 组织实施）"** 逐层展开；分功能区（功能区一/二…），每功能区先列 **建设内容清单表**（三维度：①硬件建设 ②软件建设 ③环境及配套设施），备注"现有基础迁建 / 拓展新建"，再逐项述工作原理 / 功能 / 指标 / 路线 / 场景 / 用户。
- **查重红线**：经费概算 **50 万元以上**设备 / 软件须按"院条件资源一本账"开展**查重** + 测算**预估使用率**（避免重复建设）。

### 3. 经费概算报告结构（"见事见钱"）

- **总述**：总经费 + 分项 + 测算法规依据；长周期复杂系统用**类比法 / 工程法**；**软件按 4 号文测算**（见 [[uwacomm]] 体系外的 jy-pricing skill / Pricing 项目）；全寿命周期费效分析 + 经济性分析；具备条件的加国内外同类**类比分析** + 全院**查重结论**。
- **概算总表 8 类费用**：①设备费 ②材料费 ③外部协作费 ④燃料动力费 ⑤会议/差旅/国际合作与交流费 ⑥出版/文献/信息传播/知识产权事务费 ⑦专家咨询费 ⑧其他支出（每类按功能区拆分）。
- **设备费明细**：按功能区列设备明细表（序号/名称/型号配置/单价/数量/经费），**能打包就打包**（如服务器+交换+线缆=计算环境），勿拆过细。
- **单台（套）设备四段式**：（1）主要功能用途 →（2）关键性能参数 →（3）市场报价 / 历史合同（≥3 家供应商询价，佐证附后）→（4）经费测算过程及结论（测算方法 / 取最低有效报价为市场参考价 / 参照中咨评估价 / 取二者较低或评估价为最终概算控制价）。

## 关键口径速记

- 架构话术：**"一主三分""一体两翼"**等 + 图表化；总体思路五要素（指导思想 / 原则 / 架构 / 关系 / 创新点）。
- 经费：**软件 = 4 号文**；**50 万+ 设备查重 + 预估使用率**；**利旧**单列章（现有资产迁建 vs 新建）；**中咨评估价**作概算控制价基准。
- 论证句式：国内外建设情况用"机构—年份—内容—指标—功能—效益"六要素排比；建设必要性 → 需求 → 目标 → 方案 形成闭环。
