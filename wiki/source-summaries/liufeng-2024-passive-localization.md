## 核心观点

1. **"被动" = 只收不发，是节能/隐蔽场景的定位范式** —— 论文在脚注里把"水声被动定位"定义为："水下移动节点仅根据接收的声学测量信息（信标或其他节点广播的定位报文）实现定位"。与主动 USBL/LBL 的 RTT/应答不同，被动模式下信标单向广播定位报文，节点只接收、不回复，因此作用距离更远、隐蔽性更强、节点能耗低，适合 AUV/UG 等水下移动节点的长时程运行。
2. **被动定位的核心矛盾是"测量稀疏 + 时间异步 + 模型非线性"三重耦合** —— 被动模式下节点得不到 RTT，必须用 TDoA 消除时间异步；同时水声测量频率低（广播周期可达数十秒甚至上百秒）、接收成功率不高（论文海试中 UG 的定位报文有效接收率仅 21–30%），每个定位时隙可用信息非常有限。因此论文的中心思路是**"用时空数据融合 + 因子图把稀少的测量样本榨干"**，即把节点运动信息、历史测量信息、时间同步信息、iUSBL 方位测量等统一塞进同一个估计框架，以换取定位精度和更新率。
3. **"虚拟信标网络"是贯穿三个场景的主线范式** —— 论文在信标网络、单信标、AUV 集群三个场景都引入了"虚拟信标网络"概念：把节点在不同时刻/不同位置对同一信标（或另一节点）的测量"折算"到一个等效的瞬时信标阵列，从而把时间维度上的稀疏测量转换为空间维度上的"多信标"，以便 TDoA 或 LSM 求解。这一技巧让单信标场景在理论上可以近似用多信标方法求解，是论文最具工程价值的抽象。
4. **因子图 (Factor Graph) + BP 是处理非线性 / 异步 / 多源融合的统一工具** —— 论文后三章的每个方法几乎都落在同一种结构上：构建因子图节点（状态/测量/时间同步偏置）→ 设计不同融合视角的图结构（RMF+FG、OSF+FG、多层级融合 FG）→ BP 或 LM 求解。相比 EKF/UKF，因子图避免了线性化截断误差、支持多源异构测量融合、易于扩展，论文海试中 FG 方法比 UKF 方法精度平均高 25–30%。
5. **iUSBL（逆 USBL）是集群协同定位的关键使能器** —— 传统 USBL 把阵列放在水面/主 AUV 上给水下目标/从 AUV 定位，从 AUV 只能"被动接收"主 AUV 的定位结果（延迟大），而 **iUSBL 把阵列装在从 AUV 自身上**，从 AUV 单次接收主 AUV 的定位报文就能自行算出相对方向，不依赖主 AUV 回传，实现**真正的准实时自定位**。论文把 iUSBL 的 DOA 观测作为因子接入协同定位 FG，是 USBL 在 AUV 集群场景的重要延伸。

## 主要方法

### 场景一：基于信标网络的水下移动节点声学被动定位（Chapter 3）

**系统设计**：海面 / 海底多信标网络 + GNSS 授时同步；信标按时序广播定位报文（含 ToE 时间戳、信标 ID、位置），UG/AUV 被动接收。

**方法 1：三角形网络栅格 TDoA 定位**
- 设 3 个海面信标构成等边三角形栅格（参数 a, b, α），UG 接收到 N 个信标的 ToA 后取两两差值得到 TDoA。
- 观测方程（差分消掉未知的时间偏置 Δt_bias）：
  ```
  c · (t_i - t_j) = ||p - p_i|| - ||p - p_j||,   i ≠ j
  ```
- 最小二乘 / LM 求解 UG 水平位置。
- 性能仿真：不同网格参数 (α = 30°/45°/60°)、信标位置误差 (1%–20%)、声速误差 (0.1%–5%)、ToA 测量误差 (10⁻⁴–2·10⁻³ s) 对定位误差分布的影响。

**方法 2：信息缓存 + 虚拟信标网络构建（时空数据融合 v1）**
- 当某定位时隙收到的有效 ToA 数量不足以直接解 TDoA 方程时，利用**前序时隙缓存的历史 ToA**，结合 UG 的自身运动位移，"折算"出几个**虚拟信标**（即"如果节点没动过，信标在哪里能产生相同测量"的等效信标位置）。
- 虚拟信标网络 + 实测 ToA 形成足够的 TDoA 方程，LSM 求解。

**方法 3：运动模型 + 滤波优化（时空数据融合 v2）**
- 将水声测量信息直接融入 EKF/UKF 的观测更新，状态向量为 UG 位置 + 时间同步偏置。
- 引入 UG 的 DR 运动模型（来自 DVL/DL）作为状态转移。

**方法 4：VNR-UAL-FG 框架（Virtual-Network-Reconstruction-based Underwater Acoustic Localization with a Factor graph）**
- 滑动窗口内把多个定位时隙的 ToA 重构为"虚拟信标网络"（网络拓扑随时隙动态变化）。
- 因子图节点：节点位置 x_k、时间同步偏置 δ_k；测量因子：虚拟 TDoA、运动转移、时间同步先验。
- LM 迭代求解，更新结果反馈到下一时隙。

### 场景二：基于单信标的水下移动节点声学被动定位（Chapter 4）

**核心挑战**：只有一个信标，本身无法直接 TDoA 多边定位，必须利用时间维度。

**方法 1：V-Loc（Virtual geometry Localization）**
- 利用 UG 在不同时刻对同一信标的 ToA 和 UG 自身 DR 位移，构造**虚拟信标网络**（把单信标"复制"到 UG 历史位置反投影得到的等效位置）。
- TDoA 求解粗定位，再用有效声速误差作为优化变量精细化。

**方法 2：M-Loc（Motion filter Localization）**
- UKF 近似雅可比矩阵，状态：UG 位置 + 时间同步偏置。
- 观测：ToA / TDoA + UG 运动测量。
- 在低维状态下实现定位与时间同步联合估计。

**方法 3：RMF+FG（Raw-Measurement-Fusion with a Factor Graph）**
- 滑动窗口内的**原始 ToA 测量**直接作为因子图的测量节点，与 UG 状态节点和时间同步偏置节点连接。
- 因子类型：ToA 测量因子、运动转移因子、时间同步先验因子。
- LM 求解全局优化。

**方法 4：OSF+FG（Optimized-State-Fusion with a Factor Graph）**
- 对每个定位时隙先用 UKF 得到优化后的中间状态，再把这些"优化后状态"作为因子图的测量节点。
- 优势：降低因子图规模、加快收敛；劣势：丢失部分原始信息。

**典型测量方程**（第 i 个定位时隙 ToA）：
```
τ_i = (1/c_eff) · ||p_i - p_B|| + δ_i + n_i
```
其中 p_B 为信标位置，c_eff 为有效声速，δ_i 为时间同步偏置，n_i 为高斯测量噪声。

### 场景三：水下移动节点集群的声学协同定位（Chapter 5）

**系统模式**：主从式（主 AUV 装高精度 INS，从 AUV 低精度自定位）/ 并行式（所有 AUV 自定位精度相当）。

**方法 1：信息交互 + 状态扩展的协同定位**
- 从 AUV 的状态向量扩展包含主 AUV 的状态子空间（接收到的主 AUV 位置 + 协方差）。
- 自适应噪声估计处理主 AUV 定位不确定性。
- 利用历史测量信息做联合平滑优化。

**方法 2：集成 iUSBL 的粒子 BP 协同定位**
- 从 AUV 搭载 iUSBL 阵列，可以直接测量主 AUV 相对方向（DOA）和距离，得到 3D 相对位置。
- 粒子采样代替 EKF 线性化：从 AUV 用粒子集合近似后验，BP 消息传递更新粒子权重。
- 测量野值处理：基于马氏距离 + 粒子权重的野值剔除。

**方法 3：基于因子图的多层级融合协同定位框架**
- 三个层级融合：水声测量层 + 运动测量层 + **虚拟信标网络层（主 AUV 水声测量构造）**。
- 虚拟信标网络因子：用主 AUV 的水声测量信息 + 优化后从 AUV 历史位置构建等效信标阵列 → TDoA 因子。
- 提升 AUV 集群约 15% 协同定位性能。

### 贯穿整篇的有效声速建模（Chapter 2）

- 为了避免对声速剖面 SSP 做完整声线跟踪，论文引入**有效声速 c_eff**：
  ```
  c_eff = ess(δx, D_r, D_s) = sqrt((δx)² + (D_s - D_r)²) / t
  ```
- 即把海水视为均质介质，用一个单一的等效声速代替声线弯曲建模，保留深度差和水平距离的几何关系。
- 优点：降低状态维度、避免 SSP 依赖、简化测量方程的雅可比。
- 代价：在声速剖面梯度大的场景下有系统偏置，论文用 DASS（Depth-Averaged Sound Speed）/ DACV（Depth-Averaged Current Velocity）做补偿。

### 费希尔信息矩阵 (FIM) 与 CRLB 分析

- 对水下移动节点声学定位模型建立 FIM，推导 CRLB 作为理论性能下界。
- 揭示两个主要误差源：**节点状态信息（运动测量噪声）+ 水声测量信息（ToA/TDoA 噪声、声速误差、信标位置误差）**。
- 为后续"信息融合"思路提供理论依据：两类误差均可通过多源信息融合降低。

### AUV 集群协同定位（仿真）

- 信息交互 + 状态扩展法：从 AUV 定位误差比 INS/DR **减少 70%**
- 粒子 BP + iUSBL：从 AUV 定位误差比 INS/DR **减少 80% 以上**；弱定位从 AUV 比 EKF 提升 10%
- 多层级融合 FG 框架：比粒子 BP 提升 **25%**，比 UKF 提升 **36%**
- 虚拟信标网络因子：提升 AUV 集群 **15%** 协同定位性能

### 参数敏感性关键发现

- 信标广播周期：从 10 s → 60 s，定位误差增大约 47%（单信标）；从 10 s → 85 s，平均 RMSE 翻倍（信标网络）
- 定位报文接收成功率：< 25% 时 RMSE > 150 m；25% → 40% 时 RMSE 下降 82%；> 50% 后趋于平稳
- AUV 集群广播周期：20 s → 180 s（9 倍），从 AUV 定位 RMSE 仅增大 83–84%（融合框架的鲁棒性）

## 对 Ohmybrain / USBL 项目的启发

### 对 USBL 项目的直接借鉴

1. **iUSBL 是 USBL 项目与 AUV 集群结合的关键路径**：传统 USBL 要求主节点发 / 目标响应，iUSBL 反转架构让目标节点自带阵列、被动接收，这是 USBL 项目实现"自定位"场景的核心使能技术。USBL 项目的五元圆阵可以直接作为 iUSBL 阵列部署在 AUV 上。
2. **"虚拟信标网络"解决时间维度稀疏测量**：USBL 项目若工作在单信标场景（如单一水面母船），可以借用论文的 V-Loc 思路——把载体在不同时刻对同一信标的 ToA，结合自身 DR 位移，折算成等效的多信标 TDoA，再用 USBL 阵列的 DOA 观测做强化，形成 iUSBL + 虚拟信标 TDoA 的融合定位。
3. **因子图 + LM 可替代 USBL 的传统 EKF/UKF 流程**：USBL 项目当前的定位解算链以"DOA 估计 + 声速修正 + EKF 跟踪"为主，EKF 线性化截断误差可能在远距场景下显著。改造为因子图 + LM 非线性最小二乘（借 RMF+FG 结构）可以降低这部分误差，海试数据表明提升在 25–30% 量级。
4. **有效声速 c_eff 建模是论文与丁杰博士论文的共识**：USBL 项目必须正视 SSP 变化，但完整声线跟踪在实时定位中代价大。有效声速方法（c_eff = sqrt(Δx² + ΔD²) / t）是工程化折中，USBL 项目可以在远距外场试验中采纳。
5. **时间异步 + 被动模式是 USBL 项目未来场景的必选项**：USBL 项目若要支持"隐蔽 / 低能耗"应用（如水下传感器网络、长时程 AUV），被动 TDoA 模式必须替代传统 RTT 测距。论文的 TDoA 实现可直接移植。

### 跨项目（Hub 层）价值

6. **水下移动节点定位的"三场景分类法"是跨项目的领域框架**：论文把所有水下声学定位问题归为"信标网络 / 单信标 / 集群协同"三类，这一分类可以作为 Ohmybrain 未来水声导航相关项目的上位概念骨架。
7. **因子图 + BP 是水声通信与水声定位的通用工具**：论文再次验证了 [[message-passing-algorithms]] 的通用性——不仅能处理水声通信的信道估计 / 均衡 / 译码（UWAcomm 项目），也能统一处理水声定位的多源融合。UWAcomm 的因子图设计经验可以复用到 USBL 项目。
8. **"能检测 vs 能利用"的差距——测量成功率的工程真相**：海试显示 UG 定位报文接收成功率仅 21–30%，说明水下被动定位系统永远工作在"信息饥饿"状态下。任何算法设计都必须以"测量稀疏 + 部分缺失"为前提（YAGNI 原则：不要假设高频、高成功率的测量）。
9. **DACV 补偿 DR 是 UG 节点的绝对基线**：洋流补偿让 DR 误差从 1000+ m 降到 200+ m（约 5 倍提升），任何水下定位方法不做这一步的对比都无效。Ohmybrain 下属涉及 UG/AUV 路径估计的项目都必须实现 DACV 补偿。
10. **系统广播周期 vs 节点能耗 vs 定位精度的三角权衡**：论文定量给出"广播周期 × 9 倍 → 误差 × ~2 倍"的关系，这是所有"星座 / 信标 / 广播"系统（包括 UWAnet）的工程指南——可以用精度损失换能耗节省，但超过某阈值后代价陡峭。
