## 核心观点

1. **阵型设计是 USBL 定位精度诸多影响因素中最关键的一项** —— 目标与传感器之间相对几何位置决定系统定位精度，不同阵型得到的精度差异显著。论文认为"传统平面阵 USBL 受基阵孔径小、水下环境复杂影响，定位精度受损严重"，因此基阵配置必须与算法联合优化。
2. **多阵列构型数据融合比单一阵型更可靠** —— 将五元阵（USBL5）、四元十字阵（USBL4）、三元阵（USBL012）三种基本构型的独立定位结果作为"多源观测"，通过欧氏距离中心 + 异常剔除 + LS 安装偏差校正融合，相比任一单阵型都有精度优势。这本质上是工程化的"多视角一致性检验"。
3. **五元十字阵是天然的多构型冗余基阵** —— 同一套五元硬件（S0 + 正交线阵 S1-S3 / S2-S4）同时包含 1 个五元阵、1 个四元十字阵、8 个三元阵的观测能力，硬件不增的情况下可并行求解多个独立定位结果，为融合提供素材。
4. **融合算法对经验参数 η 鲁棒** —— 参与融合的有效点数由 η（异常点剔除阈值系数）决定，但实验显示不同 η 值下定位误差均值与标准差都较平稳，"几乎不受 η 影响"。这是融合框架在工程部署中的关键可用性特征。
5. **适用场景限定：短距、窄带、直线传播假设** —— 实验条件 R = 1 km、fc = 13.5 kHz、B = 3 kHz、声速 c = 1500 m/s、"不考虑多途与声线弯曲"。方法本身不解决多径与声速剖面问题，这些被显式排除在论文讨论之外。

| 创新点 | 解决的问题 | 性质 |
|--------|-----------|------|

## 主要方法

### 1. 五元十字阵几何与多构型分解

平面五元十字阵由两条正交线阵 S1-S0-S3 和 S2-S0-S4 构成，以阵中心 S0 为原点的直角坐标：

- S0(0,0,0)、S1(D,0,0)、S2(0,D,0)、S3(-D,0,0)、S4(0,-D,0)
- D = 0.10 m（阵元间距）

同一硬件支持的基本构型：

- **USBL5**（五元阵）—— 全部 5 元
- **USBL4**（四元十字阵）—— S1,S2,S3,S4
- **USBL012** 等 8 个三元阵 —— USBL012, USBL023, USBL034, USBL041, USBL123, USBL234, USBL341, USBL412

### 2. 三种基本构型的定位方程

令目标 T 到阵元 Si 的时延为 τᵢ，τᵢⱼ = τⱼ − τᵢ，斜距 R = c·Δt，方位角 φ、俯仰角 θ。

**三元阵（式 1）**：
```
x = τ₀₁·c·R / D
y = τ₀₂·c·R / D
z = √(R² − x² − y²)
φ = arctan(y/x)
θ = arcsin(z/R)
```

**四元十字阵 USBL4（式 2）**：
```
φ = arctan((τ₁₄ − τ₁₂) / τ₁₃)
θ = arcsin((c/D)·√((τ₁₂ − τ₁₄)² + τ₁₃²))
```

**五元十字阵 USBL5（式 3）**：
```
φ = arctan((τ₀₂ − τ₀₄) / (τ₀₁ − τ₀₃))
θ = arcsin((c/D)·√((τ₀₁ − τ₀₃)² + (τ₀₂ − τ₀₄)²))
```

### 3. 本文融合方法（USBLall）

将 USBL5、USBL4、USBL012 三组初始定位结果记为三维坐标集 {Xᵢ(xᵢ, yᵢ, zᵢ)}，执行四步融合：

**步骤 1**：对每一个定位坐标 Xᵢ，计算到其他所有点的欧氏距离之和（式 4）：

```
Lᵢ = Σⱼ≠ᵢ ρ(Xᵢ, Xⱼ),  ρ(Xᵢ, Xⱼ) = √((xᵢ−xⱼ)² + (yᵢ−yⱼ)² + (zᵢ−zⱼ)²)
```

**步骤 2**：选取 Lᵢ 最小的点作为初始定位中心（式 5）：

```
C = Xᵢ,  i = argminᵢ(Lᵢ)
```

**步骤 3**：计算中心 C 到其他点 Xⱼ 的欧氏距离 L，及其平均值 CL；若某点满足 Lⱼ > η · CL（式 6）则判为异常剔除，余下即"有效初始定位点"。η 为经验参数。

**步骤 4**：对剩余有效点用最小二乘（LS）迭代校正 USBL 系统的安装偏差，取迭代结果为最终定位坐标。

### 4. 误差度量

第 i 次测量的距离误差（式 7）：

```
eᵢ = √((xᵢ − x̂ᵢ)² + (yᵢ − ŷᵢ)² + (zᵢ − ẑᵢ)²)
```

N 次测试（取 N = 360）的均值与标准差（式 8）：

```
μ = (1/N) Σᵢ eᵢ
σ = √((1/N) Σᵢ (eᵢ − μ)²)
```

异常点个数均值（式 9，N = 360 方位角观测）：

```
ξ = (1/N) Σᵢ εᵢ
```

## 对 Ohmybrain / USBL 项目的启发

### 对 USBL 项目的直接借鉴

1. **五元圆阵可复用同款多构型融合策略** —— USBL 项目的五元圆阵同样可拆成 1 个五元、多个四元/三元子阵，把每个子阵的独立定位结果做欧氏中心 + 阈值剔除 + LS 的三步融合，作为五元圆阵基线分解法（见 [[dingjie-2020-compact-usbl]]）的补充或对照。融合框架与基线分解法在算法层面可并行存在。
2. **短距窄带参数是可复刻的仿真基线** —— R = 1 km、fc = 13.5 kHz、B = 3 kHz、D = 0.10 m 是明确的试验参数集，可在 USBL 项目仿真平台里作为"期刊级对标点"：用相同参数复现 USBLall 与三种单阵型的误差对比，验证自有实现。
3. **η 不敏感是工程化可用性的关键特征** —— 任何融合/剔除算法在落地前都应该做"参数鲁棒性扫描"。本文把 η 扫描作为必检项值得借鉴：USBL 项目中的任何阈值类参数（如 Mahalanobis 剔除因子、RANSAC 内点阈值）都应配套提供类似的鲁棒性曲线。
4. **"直线传播假设"是论文的局限，也是外场试验的差异源** —— 论文结论仅在忽略多途/声线弯曲时成立。USBL 项目如需在深水或浅水多径环境下应用，必须先接入有效声速修正（见 [[dingjie-2020-compact-usbl]]）或声线跟踪模块再评估该融合方法。

### 跨项目（Hub 层）价值

5. **多观测融合作为精度杠杆的通用范式** —— "同一硬件的多构型冗余 + 一致性检验 + 鲁棒剔除 + LS 校正"这一链路可迁移到多数阵列信号处理场景：成像声纳的子孔径融合、MIMO 水声通信的多分支解调（UWAcomm 项目）、多基站 TDOA 融合等。关键是先识别"硬件里已有但未被分解利用的独立观测"。
6. **期刊论文的"轻方法"与博士论文"重方法"对比** —— 与 [[dingjie-2020-compact-usbl]]（博士论文，体系化解决阵型/声速/校准三问题）相比，本文是典型的工程期刊短文："一个明确但较窄的改进 + 扫描对比 + 鲁棒性 + 声明优势"。Hub 层建立此两类文献的对比可以帮助判断方法的成熟度与适用范围。
7. **"多阵型误差 vs SNR"曲线作为 USBL 精度评估的标准测试项** —— 任何新 USBL 算法都应至少报告这张曲线，作为 Hub 级算法性能基线。
