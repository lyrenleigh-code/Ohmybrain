# 日常工作流

## 读论文时

```
1. Zotero 一键保存（浏览器 Connector）
2. 在 Zotero 中分类到对应 Collection + 添加标签
3. 阅读 PDF，在 Zotero 中标注重点
4. 重要论文：Obsidian 中 Ctrl+Shift+Z → 生成笔记骨架
5. 补充：核心贡献、方法、关键公式、与项目的关联
6. 添加 wikilink：[[算法卡片]] [[模块笔记]] [[相关论文]]
```

## 写代码时

```
1. 模块 README 更新后 → 运行 sync_readme_index.py
2. 调试日志追加 → 添加 [[算法卡片]] [[论文笔记]] 链接
3. 新算法实现 → 检查是否有对应算法卡片，没有则创建
```

## 每周整理

```
1. 检查 0-Inbox 是否有未归类内容
2. 更新论文笔记 README 的"近期阅读"列表
3. 检查 Dataview 查询是否有异常
```

## 关键原则

- **论文必过 Zotero**：所有论文 PDF 统一由 Zotero 管理，不散落在项目目录
- **笔记必在 Obsidian**：所有思考、关联、积累都在 Obsidian 中完成
- **代码文档在项目**：README 等技术文档随代码版本控制，通过脚本同步索引
- **按需积累**：不追求覆盖率，优先录入当前开发相关的论文和算法
